Comando: pwd
Qué hace: Muestra la ruta completa del directorio actual  
Cuándo lo usé: Al inicio para verificar dónde estaba antes de crear el proyecto

Comando: ls -la
Qué hace: Lista todos los archivos (incluidos ocultos) con detalles completos  
Cuándo lo usé: Para verificar que se creó la carpeta tpl-mi-sitio y sus archivos

Comando: cd tpl-mi-sitio
Qué hace: Cambia al directorio especificado  
Cuándo lo usé: Para entrar a la carpeta del proyecto después de crearla

Comando: mkdir tpl-mi-sitio
Qué hace: Crea una nueva carpeta/directorios  
Cuándo lo usé: Para crear la carpeta principal del proyecto TP

Comando: touch index.html
Qué hace: Crea un archivo vacío (o actualiza timestamp si existe)  
Cuándo lo usé: Para crear index.html y notas-terminal.md

Comando: cat index.html
Qué hace: Muestra el contenido completo de un archivo en terminal  
Cuándo lo usé: Para verificar rápidamente que el HTML se guardó correctamente

Comando: which code
Qué hace: Muestra la ruta del ejecutable en el PATH del sistema  
Cuándo lo usé: Para confirmar que VS Code estaba instalado y accesible desde terminal

Comando: code .
Qué hace: Abre Visual Studio Code en el directorio actual  
Cuándo lo usé: Para abrir todo el proyecto en VS Code y editar los archivos

Comando: clear
Qué hace: Limpia completamente la pantalla de la terminal  
Cuándo lo usé: Durante el desarrollo para mantener la terminal ordenada y legible

Comando: live-server
Qué hace: Inicia servidor web local con recarga automática del navegador  
Cuándo lo usé: Para probar el sitio en tiempo real y verificar los estilos CSS
